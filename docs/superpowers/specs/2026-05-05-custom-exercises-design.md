# Custom exercises — design spec

**Date:** 2026-05-05  
**Status:** Approved  

## Summary

Users can create custom training exercises by entering a short free-text prompt. Groq (LLaMA 3.3 70B) expands the prompt into a full `ExerciseSpec`-compatible JSON structure, which is stored in Supabase and integrated into the weekly training schedule alongside built-in exercises.

---

## Data model

New Supabase table: `custom_exercises`

| Column | Type | Notes |
|---|---|---|
| `id` | `uuid` | PK, default `gen_random_uuid()` |
| `user_id` | `uuid` | FK → `auth.users`, not null |
| `exercise_id` | `text` | Slug, e.g. `custom_canicross_a3f2`. Unique per user. |
| `label` | `text` | Display name generated by Groq |
| `prompt` | `text` | Original user input |
| `spec` | `jsonb` | Full `ExerciseSpec` JSON |
| `active` | `boolean` | Default `true` |
| `created_at` | `timestamptz` | Default `now()` |

**RLS:** Users can only SELECT, INSERT, UPDATE, DELETE their own rows (`user_id = auth.uid()`).

---

## API

### `POST /api/training/custom`

**Request body:** `{ prompt: string }`  
**Auth:** Supabase session cookie (user_id resolved server-side via `getUser()`).

**Flow:**
1. Validate prompt (non-empty, max 300 chars).
2. Call Groq with a compact system prompt (target: < 1500 tokens total in+out) requesting a Swedish `ExerciseSpec` JSON with fields: `label`, `definition`, `ladder` (2–4 levels), `troubleshooting` (2–3 items), `guide.setup`, `guide.steps`, `guide.commonMistakes`, `guide.stopRules`. The prompt instructs Groq to include safety notes for physically demanding activities and flag age restrictions where relevant.
3. Parse and validate the JSON response. Return HTTP 422 if parsing fails.
4. Generate `exercise_id`: `custom_` + slugified `label` + `_` + 4-char random alphanumeric suffix.
5. Insert row into `custom_exercises`.
6. Return the created row.

**Error responses:**
- `400` — missing or invalid prompt
- `422` — Groq returned unparseable JSON
- `429` — Groq rate limit hit (surface a user-friendly message)
- `500` — unexpected error

## Week plan integration

In `/api/training/week`, after resolving the breed and goals, fetch all `custom_exercises` rows for the user where `active = true`. Pass their `exercise_id` and `label` as an additional section in the Groq week-plan prompt:

```
Egna övningar (inkludera om lämpligt):
- custom_canicross_a3f2: Canicross
```

**Spec lookup:** `getExerciseSpec` is synchronous and called client-side in `TrainingCard` and `ExerciseGuideSheet`. Rather than making it async, the training page pre-loads the user's active custom exercises from Supabase on mount and passes a `customSpecs: Record<string, ExerciseSpec>` prop to `TrainingCard`. `TrainingCard` passes it on to `ExerciseGuideSheet`. Both components check `customSpecs[id]` before falling back to `getExerciseSpec(id)`.

**Metrics validation:** `metrics/route.ts` currently rejects unknown exercise IDs via `getExerciseSpec`. It must be updated to also accept IDs beginning with `custom_` (verified against the user's `custom_exercises` rows in Supabase).

## UI

### 1. Create button — training view

A `+ Lägg till eget pass` button in the training view opens a modal with:
- A single textarea: "Beskriv vad du vill träna" (placeholder: "t.ex. Cykla med hunden (canicross)")
- A submit button
- Loading indicator during generation
- Error message on failure (including rate limit)

### 2. Manage custom exercises — profile view

New section in the profile page listing all custom exercises for the account:
- Each row: `label` + active/inactive toggle + delete button
- Toggle updates `active` in Supabase
- Delete removes the row (and removes the exercise from future week plans)

### 3. In training sessions

Custom exercises render identically to built-in exercises. The spec is fetched from Supabase using the `exercise_id` fallback described above. No changes to session UI components.

---

## Out of scope

- Editing a generated spec (regenerate from prompt instead)
- Sharing custom exercises between users
- Per-dog custom exercises (account-level only)
- Custom exercises in onboarding goals
